# seis-statics v0.4.0 Release Notes

Version `0.4.0` is the time-term/refraction consumer handoff release for
`seisviewer2d` migration work.

## Sign Convention

The package treats positive delay as an event arriving later than reference:

```text
delay_s > 0
applied_shift_s = -delay_s
```

Trace-shift outputs use:

```text
corrected(t) = raw(t - shift_s)
```

The final time-term trace shift is the sum of datum and residual applied shifts
plus the negative weathering delay.

## Supported Refraction Scope

This release covers the migrated public numerical core:

- 1-layer GLI / T1LST weathering workflows
- 2-layer and 3-layer T1LST thickness and weathering conversions
- multi-layer time-term solves for `v2_t1`, `v3_t2`, and `vsub_t3`
- global fixed/solved bedrock velocity and local cell V2 (`solve_cell`) modes

Applications remain responsible for SEG-Y I/O, FastAPI/Pydantic schemas, job
runtime, artifact registries, UI state, and migration orchestration.

## Compatibility

Non-breaking for consumers that import the released public package APIs.

Breaking for consumers that still import copied local modules from
`seisviewer2d/seis_statics/`: those imports must be replaced with
`seis_statics.time_term`, `seis_statics.refraction`, `seis_statics.residual`,
`seis_statics.datum`, or `seis_statics.trace_shift` before the local copy is
removed.

## Consumer Migration Prerequisite

`seisviewer2d` should depend one-way on `seis-statics` version `0.4.0`, replace
application imports of local numerical modules with package imports, and keep
SEG-Y, FastAPI/Pydantic schemas, jobs, artifacts, and UI orchestration in
`seisviewer2d`.
